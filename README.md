## рЯФР Access Model

The site ships with a **client-side shared-password gate** вАФ no backend, no user DB.

| Path | Access |
|------|--------|
| `/`, `/handbooks`, `/docs/<handbook>`, `/login` | рЯМР Public |
| `/sections/gate-cse`, `/docs/gate-cse/*`, `/dashboard` | рЯФТ Sign-in required |

- **Shared password:** defined as `SHARED_PASSWORD` in [`app/src/lib/auth.tsx`](app/src/lib/auth.tsx). To rotate вЖТ edit the constant + redeploy.
- **Session:** stored in `localStorage` under keys `learning:auth`, `learning:display-name`, `learning:logged-in-at`.
- **Sign-in flow:** display name + shared password вЖТ dashboard (or `?next=<path>` if the user came from a gated link).

## рЯО® Design System (Porhi warm-sand palette)

Editorial feel вАФ large Fraunces serif headlines with amber italic emphasis, warm sand bodies, calm hairlines.

**Color tokens** (defined in `app/src/index.css`, exposed via Tailwind in `tailwind.config.js`):

| Group | Tokens |
|-------|--------|
| Surfaces | `sand`, `sand-2`, `sand-3`, `surface`, `surface-2` |
| Text | `ink`, `ink-2`, `ink-3`, `ink-4`, `ink-5` |
| Borders | `line`, `line-2` |
| Brand | `amber`, `amber-50..700` |
| Accents | `clay`, `sage`, `ink-blue` |

**Fonts:** `font-serif` Fraunces ¬Ј `font-sans` Geist ¬Ј `font-mono` Geist Mono ¬Ј `font-bengali` Tiro Bangla.

**Utility classes:** `.serif`, `.bn`, `.mono-font`, `.dots`, `.hairline`, `.chip[-amber/-ink/-outline]`, `.btn[-primary/-amber/-ghost/-sm/-lg]`, `.card-surface`, `.meta`.

**Dark mode:** warm-espresso counterpart with brighter amber accents; toggle via header button.

## рЯЪА Local Development

```bash
cd app

# Install (first time)
npm install --legacy-peer-deps

# Dev server (auto-runs convert script first)
npm run dev

# Production build
npm run build
```

**What the prebuild step does:**

1. `npm run convert` вАФ reads `handnote/**/*.md`, writes:
   - `app/src/generated/content/**/*.mdx`
   - `app/src/generated/content-meta.json`
2. `npm run sitemap` вАФ writes:
   - `app/public/sitemap.xml`
   - `app/public/robots.txt`

## рЯУЭ Content Authoring Flow

1. Write/edit a `.md` file under `handnote/`
2. Run `npm run dev` (or `npm run build`) in `app/`
3. The convert script:
   - Transforms ` ```mermaid ` blocks into `<Mermaid>` JSX
   - Escapes `{`, `}`, stray `<` for MDX safety
   - Rewrites `.md` links to `/docs/вА¶` routes
   - Writes `.mdx` + updates `content-meta.json`
4. Vite picks up the new .mdx as a **lazy-loaded route chunk** (no runtime parsing)
5. `handnote/<slug>.md` вЖТ **article layout** (public)
   `handnote/gate-cse/<NN>-<slug>.md` вЖТ **course layout** (sidebar + Prev/Next, sign-in required)

## рЯ§Ц AI Skills & Agents

The project ships with ready-to-use Claude Code skills and a specialist agent under `.claude/` to speed up common workflows.

### Skills (invoke with `/<name>` in Claude Code)

| Skill | Purpose |
|-------|---------|
| `/add-chapter` | Author a new GATE CSE chapter under `handnote/gate-cse/` following the 9-section template |
| `/new-handbook` | Author a new standalone handbook under `handnote/` with WhatвЖТWhyвЖТHow flow |
| `/build-check` | Run production build and report bundle sizes / warnings (no deploy) |
| `/docker-run` | Build the Docker image and run on `localhost:8080` with smoke tests (offline preview / pre-deploy validation) |
| `/content-audit` | Lint all `handnote/*.md` for compliance (frontmatter, Mermaid, code fences, broken links) вАФ reports only, no auto-fix |
| `/deploy` | Build + ship to Cloudflare Pages + smoke-test live URL |
| `/ui-test` | End-to-end Playwright MCP tests on the deployed site |

### Agents

| Agent | Invocation | Purpose |
|-------|-----------|---------|
| `content-writer` | `Agent` tool with `subagent_type: content-writer` | Drafts / expands `.md` content following project voice, templates, Mermaid + KaTeX conventions. Pure authoring вАФ no code or deploy. |
| `app-engineer` | `Agent` tool with `subagent_type: app-engineer` | Writes / modifies React + TS code under `app/src/` вАФ knows Porhi tokens, layout taxonomy, auth gate, MDX pipeline. Scoped to `app/`; never touches `handnote/`. |

Skill / agent definitions live in `.claude/skills/*.md` and `.claude/agents/*.md` respectively.
