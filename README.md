**Version:** 1.0.0

## 🔐 Access Model

The site ships with a **client-side shared-password gate** — no backend, no user DB.

| Path | Access |
|------|--------|
| `/`, `/handbooks`, `/docs/<handbook>`, `/login` | 🌐 Public |
| `/sections/gate-cse`, `/docs/gate-cse/*`, `/dashboard` | 🔒 Sign-in required |

- **Shared password:** defined as `SHARED_PASSWORD` in [`app/src/lib/auth.tsx`](app/src/lib/auth.tsx). To rotate → edit the constant + redeploy.
- **Session:** stored in `localStorage` under keys `learning:auth`, `learning:display-name`, `learning:logged-in-at`.
- **Sign-in flow:** display name + shared password → dashboard (or `?next=<path>` if the user came from a gated link).

## 🎨 Design System (Porhi warm-sand palette)

Editorial feel — large Fraunces serif headlines with amber italic emphasis, warm sand bodies, calm hairlines.

**Color tokens** (defined in `app/src/index.css`, exposed via Tailwind in `tailwind.config.js`):

| Group | Tokens |
|-------|--------|
| Surfaces | `sand`, `sand-2`, `sand-3`, `surface`, `surface-2` |
| Text | `ink`, `ink-2`, `ink-3`, `ink-4`, `ink-5` |
| Borders | `line`, `line-2` |
| Brand | `amber`, `amber-50..700` |
| Accents | `clay`, `sage`, `ink-blue` |

**Fonts:** `font-serif` Fraunces · `font-sans` Geist · `font-mono` Geist Mono · `font-bengali` Tiro Bangla.

**Utility classes:** `.serif`, `.bn`, `.mono-font`, `.dots`, `.hairline`, `.chip[-amber/-ink/-outline]`, `.btn[-primary/-amber/-ghost/-sm/-lg]`, `.card-surface`, `.meta`.

**Dark mode:** warm-espresso counterpart with brighter amber accents; toggle via header button.

## 🚀 Local Development

```bash
cd app

# Install (first time)
npm install --legacy-peer-deps

# Dev server (auto-runs convert script first)
npm run dev

# Production build
npm run build
```

**What the prebuild step does:**

1. `npm run convert` — reads `handnote/**/*.md`, writes:
   - `app/src/generated/content/**/*.mdx`
   - `app/src/generated/content-meta.json`
2. `npm run sitemap` — writes:
   - `app/public/sitemap.xml`
   - `app/public/robots.txt`

## 🐳 Docker Deployment

```bash
docker build -t learning-hub:latest .
docker run -d -p 8080:80 learning-hub:latest
```

Multi-stage build: Node for the React build, `nginx:alpine` for serving. Nginx is hardened with security headers, gzip, and immutable caching for fingerprinted assets.

## 📝 Content Authoring Flow

1. Write/edit a `.md` file under `handnote/`
2. Run `npm run dev` (or `npm run build`) in `app/`
3. The convert script:
   - Transforms ` ```mermaid ` blocks into `<Mermaid>` JSX
   - Escapes `{`, `}`, stray `<` for MDX safety
   - Rewrites `.md` links to `/docs/…` routes
   - Writes `.mdx` + updates `content-meta.json`
4. Vite picks up the new .mdx as a **lazy-loaded route chunk** (no runtime parsing)
5. `handnote/<slug>.md` → **article layout** (public)
   `handnote/gate-cse/<NN>-<slug>.md` → **course layout** (sidebar + Prev/Next, sign-in required)

## 🤖 AI Skills & Agents

The project ships with ready-to-use Claude Code skills and a specialist agent under `.claude/` to speed up common workflows.

### Skills (invoke with `/<name>` in Claude Code)

| Skill | Purpose |
|-------|---------|
| `/add-chapter` | Author a new GATE CSE chapter under `handnote/gate-cse/` following the 9-section template |
| `/new-handbook` | Author a new standalone handbook under `handnote/` with What→Why→How flow |
| `/build-check` | Run production build and report bundle sizes / warnings (no deploy) |
| `/deploy` | Build + ship to Cloudflare Pages + smoke-test live URL |
| `/ui-test` | End-to-end Playwright MCP tests on the deployed site |

### Agents

| Agent | Invocation | Purpose |
|-------|-----------|---------|
| `content-writer` | `Agent` tool with `subagent_type: content-writer` | Drafts / expands `.md` content following project voice, templates, Mermaid + KaTeX conventions. Pure authoring — no code or deploy. |

Skill / agent definitions live in `.claude/skills/*.md` and `.claude/agents/*.md` respectively.
