## 💻 21.2 Structure Padding & Alignment — Interview Favorite
**sizeof trap:**
```c
struct A { char c; int i; char d; }; 
// sizeof != 1+4+1 (6)
// Actual: 1 + 3(pad) + 4 + 1 + 3(pad) = 12!
```
মেমোরি একসেস ফাস্ট করার জন্য কম্পাইলার ডাটার মাঝে কিছু ফাকা জায়গা বা **Padding** যোগ করে। মেমোরি সেভ করতে বড় মেম্বারগুলো আগে রাখতে হয়।

# 🇧🇩 Job Exam Special (Bangladesh Context)
### Problem 01: Sorting an array of Structures (Student GPA)
**পরীক্ষা:** বিপিএসসি - এ্যাসিস্ট্যান্ট প্রোগ্রামার (২০২১)
<details>
<summary><b>💻 Solution in C</b></summary>
```c
#include `stdio.h`
#include `string.h`

struct Student {
    char name[20];
    float gpa;
};

void sort(struct Student s[], int n) {
    for(int i=0; i<n-1; i++)
        for(int j=0; j<n-i-1; j++)
            if(s[j].gpa < s[j+1].gpa) {
                struct Student temp = s[j];
                s[j] = s[j+1];
                s[j+1] = temp;
            }
}
```
</details>

### 🔵 Group B: Written Questions (Solutions included)

**Q1: Structure এবং Union এর মধ্যে ৪টি মূল পার্থক্য লিখুন।**
**উত্তরঃ** 
1. **Memory:** Struct-এ সব মেম্বার আলাদা মেমোরি পায়। Union-এ সব মেম্বার একই মেমোরি শেয়ার করে।
2. **Size:** Struct-এর সাইজ সব মেম্বারের সাইজের যোগফলের বেশি বা সমান। Union-এর সাইজ তার সবচেয়ে বড় মেম্বারের সমান।
3. **Usage:** Struct-এ একসাথে সব মেম্বারে ডাটা রাখা যায়। Union-এ একবারে শুধু একটি মেম্বারে ভ্যালিড ডাটা থাকে।
4. **Keyword:** এখানে `struct` এবং ওখানে `union` ব্যবহৃত হয়।

**Q2: কেন struct মেম্বার এক্সেস করার সময় 'Arrow (->)' অপারেটর দরকার হয়? উদাহরন দাও।**
**উত্তরঃ** যখন আমরা স্ট্রাকচারকে রেফারেন্স বা পয়েন্টার দিয়ে পাস করি, তখন ওই পয়েন্টারটি সরাসরি মেম্বারকে চেনে না। পয়েন্টারকে ডিরেফারেন্স করে মেম্বার চেনার শর্টকাট হলো `->` অপারেটর।
```c
struct Point { int x, y; };
struct Point p1 = {10, 20};
struct Point *ptr = &p1;
printf("%d", ptr->x); // shorthand for (*ptr).x
```

**Q3: Memory Alignment এবং Structure Padding কেন প্রয়োজন?**
**উত্তরঃ** আধুনিক প্রসেসরগুলো মেমোরি থেকে ডাটা পড়ার সময় ওয়ার্ড বা ব্লক আকারে পড়ে (সাধারণত ৪ বা ৮ বাইট)। যদি ডাটা এলোমেলো বা মিস-এলাইনড থাকে, তবে প্রসেসরকে একবারের জায়গায় দুবার মেমোরি রিড করতে হয়, যা সিস্টেমকে স্লো করে দেয়। প্যাডিং ব্যবহার করে ডালাকে বাউন্ডারিতে এলাইন করলে এক্সেস স্পিড কয়েক গুণ বেড়ে যায়।

**Q4: Nested Structure বলতে কী বুঝো? একটি বাস্তব উদাহরণ দাও।**
**উত্তরঃ** যখন একটি স্ট্রাকচারের ভেতর অন্য একটি স্ট্রাকচার মেম্বার হিসেবে থাকে, তাকে Nested Structure বলে।
```c
struct Date { int day, month, year; };
struct Employee {
    int id;
    struct Date dob; // Nested struct
};
```
এটি ডাটাকে আরো অর্গানাইজড এবং হায়ারার্কিকাল করতে সাহায্য করে।

**Q5: Bit-fields ব্যবহারের একটি সীমাবদ্ধতা উল্লেখ করুন।**
**উত্তরঃ** Bit-fields এর সবচেয়ে বড় অসুবিধা হলো এটি **Machine Dependent**। বিভিন্ন কম্পাইলার বা আর্কিটেকচার বিটগুলোকে ভিন্নভাবে সাজাতে পারে (Endianness অনুযায়ী)। এছাড়া বিট-ফিল্ড ই এলিমেন্টের অ্যাড্রেস (`&` operator) নেওয়া যায় না।

## 📝 Summary
- **Struct:** Custom multi-type container. Memory = sum of members + padding.
- **Union:** Memory sharing container. Size = size of the largest member.
- **Padding:** Extra bytes added by compiler for CPU speed (alignment).
- **Arrow (->):** Used to access struct members via a pointer.
- **Bit-fields:** Packing data into bits to save memory.
- **Typedef:** Creating aliases for complex types.
