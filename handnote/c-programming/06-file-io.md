# Topic 22: File Handling (Text & Binary)
### Problem 01: Copy one file to another in C
**পরীক্ষা:** ব্যাংক আইটি (পুবালী ব্যাংক ২০২৩)
```c
#include `stdio.h`
int main() {
    FILE *src = fopen("in.txt", "r"), *dst = fopen("out.txt", "w");
    char ch;
    if(src == NULL || dst == NULL) return 1;
    while((ch = fgetc(src)) != EOF) fputc(ch, dst);
    fclose(src); fclose(dst);
    return 0;
}
```
### Problem 02: Command Line Arguments (Sum of two numbers)
**বিবরণ:** argc এবং argv ব্যবহার করে ইনপুট নেওয়া।
```c
#include `stdio.h`
#include `stdlib.h`
int main(int argc, char *argv[]) {
    if(argc < 3) {
        printf("Usage: %s num1 num2\n", argv[0]);
        return 1;
    }
    printf("Sum: %d", atoi(argv[1]) + atoi(argv[2]));
    return 0;
}
```

## 📖 24.1 fseek(), ftell(), and rewind()
| Function | Signature | Purpose |
|----------|-----------|---------|
| fseek() | int fseek(FILE *stream, long offset, int whence) | নির্দিষ্ট পজিশনে কার্সর মুভ করে। |
| ftell() | long ftell(FILE *stream) | বর্তমান পয়েন্টার পজিশন নম্বর দেয়। |
| rewind() | void rewind(FILE *stream) | পয়েন্টারকে একবারে শুরুতে নিয়ে যায়। |

**Whence Parameters:**
- SEEK_SET: ফাইলের শুরু থেকে।
- SEEK_CUR: বর্তমান পজিশন থেকে।
- SEEK_END: ফাইলের শেষ থেকে।

### 💻 Code: Find File Size
```c
FILE *fp = fopen("data.txt", "r");
if(fp == NULL) return 1;
fseek(fp, 0, SEEK_END);          // শেষে যাও
long size = ftell(fp);           // পজিশনই হলো সাইজ
printf("File Size: %ld bytes", size);
fclose(fp);
```

# 🕵️‍♂️ Dynamic Memory Internals (Deep)
### Memory Leak vs Dangling Pointer
- **Memory Leak:** malloc করেছেন কিন্তু free করেননি। মেমোরি ব্লকটি হারিয়ে গেছে।
- **Dangling Pointer:** free করেছেন কিন্তু পয়েন্টারটিকে NULL করেননি। সেটি এখনও একটি মৃত এড্রেসকে পয়েন্ট করছে।
** ডিবাগিং টিপ:** লিনাক্সে মেমোরি লিগ চেক করার জন্য **Valgrind** টুল ব্যবহার করা হয়। (Command: valgrind --leak-check=full ./a.out)

### 🔵 Group B: Written Questions (Solutions included)

**Q1: টেক্সট ফাইল এবং বাইনারি ফাইলের মধ্যে ৩টি প্রধান পার্থক্য লিখুন।**
**উত্তরঃ** 
1. **Readable:** টেক্সট ফাইল মানুষ নোটপ্যাড দিয়ে পড়তে পারে। বাইনারি ফাইল মানুষের পড়ার উপযোগী নয়।
2. **Storage:** টেক্সট ফাইলে ডাটা ASCII ক্যারেক্টার হিসেবে থাকে। বাইনারি ফাইলে ০ এবং ১ এর সিকোয়েন্স হিসেবে থাকে।
3. **newline:** টেক্সট ফাইলে `\n` কে সিস্টেম ভেদে (`\r\n` বা `\n`) কনভার্ট করা হয়। বাইনারি ফাইলে কোনো কনভার্সন হয় না।

**Q2: `fprintf()` এবং `fwrite()` এর মধ্যে কখন কোনটি ব্যবহার করা উচিত?**
**উত্তরঃ** 
- **`fprintf()`:** যখন আপনি ডালাকে মানুষের পড়ার উপযোগী ফরমেটে (যেমন: স্পেস বা কমা দিয়ে আলাদা করে) টেক্সট ফাইলে সেভ করতে চান, তখন এটি শ্রেয়। এটি কিছুটা স্লো কারণ এতে ফরম্যাটিং এর কাজ থাকে।
- **`fwrite()`:** যখন আপনি মেমোরির ডাটা (যেমন: একটি পুরো স্ট্রাকচার বা অ্যারে) হুবহু ফাইলে ডাম্প করতে চান, তখন এটি ব্যবহার করা হয়। এটি অত্যন্ত দ্রুত এবং বাইনারি ফাইলের জন্য পারফেক্ট।

**Q3: কমান্ড লাইন আর্গুমেন্ট ব্যবহার করার সুবিধা কী?**
**উত্তরঃ** এটি ইউজারকে প্রোগ্রাম রান করার সময় ইনপুট দেওয়ার সুযোগ দেয়। ফলে প্রোগ্রামের ভেতর বারবার `scanf()` দিয়ে ইনপুট দিতে হয় না। এছাড়া স্ক্রিপ্টিং বা অটোমেশনের সময় এক কমান্ড থেকে অন্য কমান্ডে ডাটা পাস করতে এটি অপরিহার্য।

**Q4: ফাইল হ্যান্ডলিং এ `fclose()` কেন অত্যন্ত গুরুত্বপূর্ণ?**
**উত্তরঃ** 
1. **Data Integrity:** অনেক সময় ডাটা বাফারে থেকে যায়, যা ফাইল ক্লোজ না করলে হার্ডডিস্কে রাইট হয় না।
2. **Resource Leak:** অপারেটিং সিস্টেমের একটি নির্দিষ্ট লিমিট থাকে যে একটি প্রোগ্রাম কয়টি ফাইল ওপেন রাখতে পারবে। ফাইল ক্লোজ না করলে মেমোরি এবং ফাইল হ্যান্ডেল লিক হয়।
3. **Locking:** অনেক সময় ফাইল ওপেন থাকলে অন্য প্রোগ্রাম সেটি এডিট করতে পারে না।

**Q5: নিচের কোডের সমস্যা কী এবং সমাধান কী?**
```c
#define MAX 10
void func() {
    #undef MAX
    printf("%d", MAX);
}
```
**উত্তরঃ** 
- **সমস্যা:** কোডটি কম্পাইল এরর দিবে। কারণ `func()` এর ভেতর `MAX` কে ব্যবহার করার আগেই `#undef` দিয়ে সেটিকে মুছে ফেলা হয়েছে। ফলে `printf` এ যখন `MAX` কে পাওয়া যাবে, কম্পাইলার সেটিকে চিনবে না।
- **সমাধান:** যদি আপনি আন-ডিফাইন করতে চান, তবে সেটি ব্যবহারের পরে করা উচিত অথবা আন-ডিফাইন করার বদলে ভ্যালু আপডেট করা উচিত।
