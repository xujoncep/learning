## 🧠 3. Core Concepts

### 3.1 Number Systems

#### Representations

| Decimal | Binary | Octal | Hex |
|---------|--------|-------|-----|
| 10 | 1010 | 12 | A |
| 15 | 1111 | 17 | F |
| 255 | 11111111 | 377 | FF |

#### Signed Number Representations

For n bits:

| System | Range |
|--------|-------|
| **Sign-Magnitude** | -(2^(n-1)-1) to +(2^(n-1)-1) |
| **1's Complement** | -(2^(n-1)-1) to +(2^(n-1)-1) |
| **2's Complement** | -2^(n-1) to +(2^(n-1)-1) |

**Why 2's complement?** Only one zero, arithmetic consistent।

#### 2's Complement

**To find 2's complement:** Invert bits + add 1।

**Example:** +5 = 0101 (4-bit)
- 1's complement: 1010
- 2's complement: 1011 = -5

---

### 3.2 Floating Point (IEEE 754)

#### Single Precision (32 bits)

| Sign | Exponent | Mantissa |
|------|----------|----------|
| 1 bit | 8 bits | 23 bits |

- **Bias:** 127
- **Value:** `(-1)^S × 1.M × 2^(E-127)`

#### Example

25.375 in IEEE 754:
- Binary: `11001.011`
- Normalize: `1.1001011 × 2⁴`
- Exponent: 4+127 = 131 = 10000011
- Mantissa: 1001011 (23 bits total with trailing 0s)
- Sign: 0

Result: `0 10000011 10010110000000000000000`

---

### 3.3 Addressing Modes

| Mode | Description | Example |
|------|-------------|---------|
| **Immediate** | Operand in instruction | `MOV R1, #5` |
| **Direct** | Address in instruction | `MOV R1, [100]` |
| **Indirect** | Address of address | `MOV R1, [[100]]` |
| **Register** | Register holds operand | `MOV R1, R2` |
| **Register indirect** | Register holds address | `MOV R1, [R2]` |
| **Indexed** | Base + Index | `MOV R1, [R2+5]` |
| **Relative** | PC + offset | Branch instructions |

---

### 3.4 Instruction Pipelining

**Without pipelining:** Each instruction takes k cycles। N instructions = N×k cycles।

**With k-stage pipelining:** N instructions = (k + N - 1) cycles।

**Speedup:** `S = N×k / (k + N - 1)` → approaches k as N increases।

#### Pipeline Hazards

1. **Structural hazard** — resource conflict
2. **Data hazard** — instruction depends on earlier instruction's result
3. **Control hazard** — branch instructions

#### Data Hazard Types

- **RAW (Read After Write)** — true dependency
- **WAR (Write After Read)** — anti-dependency
- **WAW (Write After Write)** — output dependency

**Solutions:** Forwarding, pipeline stall, reordering

### 3.5 Cache Memory

#### Why Cache?

Memory এর speed CPU এর চেয়ে slow। Frequently used data cache এ রেখে speed বাড়ানো হয়।

#### Memory Hierarchy

```
Register → L1 Cache → L2 Cache → L3 Cache → RAM → Disk
Fast/Small                              Slow/Large
```

#### Cache Mapping

##### 1. Direct Mapping

Each memory block maps to **exactly one** cache line।

`Cache line = (Block address) mod (Number of cache lines)`

##### 2. Fully Associative

Any block can go to **any** cache line। Expensive lookup।

##### 3. Set Associative (n-way)

Cache divided into sets, each set has n lines। Block maps to specific set, any line within।

#### Address Breakdown

For cache of `C` lines, block size `B`, memory `M`:

**Direct mapping:**
- Offset = log₂(B)
- Line = log₂(C)
- Tag = log₂(M) - log₂(C) - log₂(B)

**n-way set associative:**
- Sets = C/n
- Set index = log₂(C/n)
- Tag = remaining bits

#### Hit and Miss

- **Hit rate (h):** fraction of accesses found in cache
- **Miss rate (1-h)**

#### Average Memory Access Time (AMAT)

`AMAT = Hit_time + Miss_rate × Miss_penalty`

Or for multi-level:
`AMAT = h × T_cache + (1-h) × T_mem`

---

### 3.6 Main Memory

#### Types

- **DRAM** — slower, cheaper, needs refresh
- **SRAM** — faster, expensive, used in cache

#### Memory Interleaving

Memory banks parallel access। k-way interleaving → access k words in 1 memory cycle।

---

### 3.7 I/O Interface

#### Modes

1. **Programmed I/O** — CPU polls, slow
2. **Interrupt-driven I/O** — Device interrupts CPU
3. **DMA (Direct Memory Access)** — Device transfers directly to memory, CPU free

## 📐 4. Formulas & Shortcuts

### Number System Conversions

- Binary to Decimal: positional sum
- Decimal to Binary: repeated division by 2
- Binary to Hex: group 4 bits from right

### Floating Point

- Single precision bias = 127
- Double precision bias = 1023

### Pipelining

- Speedup S = k / (1 + (k-1)/N)
- Throughput = N / (k+N-1) × clock rate
- Ideal speedup = k (number of stages)

### Cache

- Block = memory bytes transferred at once
- Cache size = Number of lines × Block size
- AMAT = h × t_hit + (1-h) × t_miss

### Memory Access

- `Effective Access Time (EAT)` = h × t_cache + (1-h) × t_memory

### 🔹 Pipelining Questions

#### PYQ 6 (GATE 2024) — Speedup

5-stage pipeline, 100 instructions। Speedup over non-pipelined?

**Solution:**
Non-pipelined: 100 × 5 = 500 cycles
Pipelined: 5 + 99 = 104 cycles
Speedup = 500 / 104 ≈ **4.81** ✅

#### PYQ 8 (GATE 2022) — Pipeline Cycles

4-stage pipeline, stages take 10, 8, 10, 12 ns. 10 instructions total time?

**Solution:**
Max stage time = 12 ns (clock period)
Total time = (4 + 10 - 1) × 12 = 13 × 12 = **156 ns** ✅

### 🔹 Cache Questions

#### PYQ 10 (GATE 2024) — Cache Mapping

16 KB direct-mapped cache, 32-byte blocks, 32-bit address। Tag bits?

**Solution:**
- Block offset = log₂(32) = **5 bits**
- Number of lines = 16 KB / 32 B = 512 = 2^9 → **9 bits** index
- Tag = 32 - 5 - 9 = **18 bits** ✅

---

#### PYQ 11 (GATE 2023) — Set Associative

32 KB, 4-way set associative, 64 B blocks। Set index bits?

**Solution:**
- Lines = 32 KB / 64 = 512
- Sets = 512 / 4 = 128 = 2^7 → **7 bits** ✅

#### PYQ 12 (GATE 2022) — AMAT

Cache hit time 2 ns, miss penalty 50 ns, hit rate 95%। AMAT?

**Solution:**
AMAT = 2 + 0.05 × 50 = 2 + 2.5 = **4.5 ns** ✅

#### PYQ 13 (GATE 2021) — Hit Rate

Memory access time 100 ns, cache 10 ns, hit rate 0.9। EAT?

**Solution:**
EAT = 0.9 × 10 + 0.1 × (10 + 100) = 9 + 11 = **20 ns** ✅

#### PYQ 14 (GATE 2020) — Cache Size

Cache size 64 KB, block 32 B। Lines?

64 KB / 32 B = **2048 lines** ✅

## ⚠️ 8. Traps & Common Mistakes

- ❌ **Byte vs Word** addressable — read carefully
- ❌ **Cache line = block**, often confused
- ❌ **Direct mapped: line = block mod C**, not block / C
- ❌ **Pipelining clock = slowest stage**, not average
- ❌ **First instruction doesn't get speedup** — only subsequent
- ❌ **Branch taken/not taken** — both need considered for CPI
- ❌ **AMAT formula** — miss penalty includes cache access sometimes
- ❌ **2's complement range asymmetric:** -128 to +127 (not -127)
- ❌ **Register indirect vs Indirect** — different modes
- ❌ **RISC = fewer instructions, simpler, fixed length**

## 📝 9. Quick Revision Summary

### Must-Remember Formulas

```
Pipeline speedup = N×k / (k+N-1)
AMAT = hit_time + miss_rate × miss_penalty
EAT = h × t_cache + (1-h) × t_memory
CPI = 1 + Σ(freq × stall)

Direct mapped cache bits:
  offset = log₂(block_size)
  index = log₂(num_lines)
  tag = addr_bits - offset - index

Set associative:
  sets = num_lines / ways
  index = log₂(sets)
```

### Key Numbers

- IEEE 754 single: 1+8+23, bias 127
- IEEE 754 double: 1+11+52, bias 1023
- 8-bit signed range: -128 to +127
- 16-bit signed: -32768 to +32767
- 32-bit unsigned: 0 to 4,294,967,295

### Addressing Modes Summary

| Mode | Access |
|------|--------|
| Immediate | operand in instr |
| Direct | memory[addr] |
| Indirect | memory[memory[addr]] |
| Register | register content |
| Indexed | register + offset |
